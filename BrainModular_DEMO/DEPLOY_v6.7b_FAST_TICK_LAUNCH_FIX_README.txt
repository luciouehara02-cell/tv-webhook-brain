BrainRAY / BrainModular v6.7b FAST_TICK_LAUNCH_FIX
=====================================================

Base:
- v6.7a TEST2 first-entry filter package.

Why this version exists:
- Full replay showed the first-entry TEST2 filter worked correctly.
- The weak part was FAST_TICK_LAUNCH: it produced two poor replay trades:
  - tick_confirmed_launch_long -1.0681%
  - tick_confirmed_launch_long -0.3391%

Code fix added in v6.7b:
1. Fast-tick launch expiry now uses event-time when REPLAY_USE_EVENT_TIME_FOR_POSITION_CLOCK=true.
   This removes the replay-compression issue where a 45-second live window could cover many market minutes in replay.
2. Fast-tick launch now arms only from a healthy feature context:
   - RSI >= FAST_TICK_LAUNCH_ARM_MIN_RSI
   - ADX >= FAST_TICK_LAUNCH_ARM_MIN_ADX
   - close >= EMA8
   - EMA8 >= EMA18
   - FVVO score is non-negative
3. Fast-tick launch re-checks EMA/FVVO context again at tick confirmation before entry.
4. Blocked launch arms are logged as FAST_TICK_LAUNCH_NOT_ARMED with the exact reason.
