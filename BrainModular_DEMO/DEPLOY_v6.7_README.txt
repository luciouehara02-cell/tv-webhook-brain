BrainRAY / BrainModular v6.7 EXIT_RETRY_PROFIT_PROTECT

Base: v6.6e FIX4A balanced/optimal.

Code changes:
1. Exit webhook retry / pending-exit protection.
   - If exit_long forward to 3Commas fails with 503 / timeout / network error, the Brain keeps the position logically open.
   - New entries are blocked while pendingExit is active.
   - Exit retries are attempted using EXIT_FORWARD_RETRY_DELAYS_MS.
   - The Brain marks itself flat only after 3Commas forward returns OK, or immediately when ENABLE_HTTP_FORWARD=false for replay.

2. Status/snapshot visibility.
   - /status includes pendingExit and exitForwardRetryConfig.
   - startup CONFIG_SNAPSHOT includes exitForwardRetry.

Recommended v6.7 variables:
- EXIT_FORWARD_RETRY_ENABLED=true
- EXIT_FORWARD_RETRY_DELAYS_MS=0,2000,5000,15000
- EXIT_FORWARD_RETRY_BLOCK_ENTRIES=true
- BREAKEVEN_OFFSET_PCT=0.20
- DYNAMIC_BE_NORMAL_OFFSET_PCT=0.20
- FIRST_ENTRY_NO_PROGRESS_MIN_BARS=5
- FIRST_ENTRY_THESIS_FAIL_RSI_BELOW=52

npm install --no-audit --no-fund && node server.js
